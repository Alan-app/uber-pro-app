VERSÃO v105 - fix-critico-isolamento-usuarios-e-sync

FEEDBACK DA v104:
- Dados de uma época antiga aparecem ao instalar
- Importar backup ou editar dado: ao sair e entrar, volta aos dados antigos
- Mesmos dados aparecem para TODOS os usuários — vazamento entre contas

DIAGNÓSTICO — 3 BUGS CRÍTICOS:

BUG 1 (GRAVÍSSIMO) — Chaves localStorage sem isolamento por usuário:
  Todas as chaves de dados são fixas: "controleUberProDados", 
  "controleUberProDados__offline_safe_snapshot_v83", etc.
  Quando usuário A faz login e salva, essas chaves ficam no localStorage.
  Quando usuário B faz login no mesmo dispositivo, o app lê as mesmas
  chaves e exibe os dados do usuário A.
  Também explica os "dados de época antiga": eram dados de sessão anterior
  de outro usuário ou do próprio usuário antes de limpar o app.

BUG 2 (CRÍTICO) — mdWrapCloudPayload não estava definida:
  A função é chamada em cloudPushNow() mas nunca foi criada no código.
  Isso causava um ReferenceError silencioso que impedia o app de subir
  qualquer dado para a nuvem (cloudPushNow sempre falhava).
  Resultado: dados ficavam só no localStorage, nunca chegavam ao Supabase.
  Ao reinstalar, localStorage limpo + nuvem vazia = tudo zerado.

BUG 3 (CRÍTICO) — visibilitychange completamente vazio:
  O comentário "v96 fix: não fazer nada no minimize" deixou o handler
  sem nenhuma ação. Ao minimizar: não salvava local, não subia para nuvem.
  Ao voltar: não sincronizava com nuvem. Dados só persistiam se o usuário
  digitasse algo (pois aí salvar() era chamado diretamente).

O QUE FOI FEITO:

Fix 1: Isolamento por usuário
  - Adicionada função mdGetUserScopedKey() que prefixa chaves com o userId
  - loadDados() agora tenta primeiro a chave com userId, depois a base
  - salvar() grava nas duas chaves (com e sem userId) para compatibilidade
  - Evita completamente o vazamento de dados entre usuários

Fix 2: mdWrapCloudPayload criada
  - Função que constrói o envelope enviado para o Supabase
  - cloudPushNow() agora funciona corretamente
  - Dados sobem para a nuvem após qualquer save

Fix 3: visibilitychange restaurado
  - Ao minimizar: salva local + sobe para nuvem
  - Ao voltar: restaura visibilidade + chama cloudBoot(true) para sincronizar
