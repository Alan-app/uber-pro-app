MetaDriver vBeta1.9

Solicitado:
- Em Metas (seção "Configurar meta"), no celular, ao tocar em "Avançar" no teclado virtual após digitar a Descrição, o foco ia para o Valor; após digitar o Valor, ia para a Descrição da meta seguinte. O esperado é que o teclado finalize a edição (como um OK), sem avançar para outro campo.

Causa:
- A correção da vBeta1.6 tratava a tecla Tab, que funciona em teclado físico. No celular, o botão "Avançar" do teclado virtual não dispara essa tecla, então a correção não tinha efeito nesse cenário.

Feito:
- Adicionado enterkeyhint="done" aos campos Descrição e Valor das metas, fazendo o teclado virtual exibir "Concluído" em vez de "Avançar".
- O botão "Concluído" agora finaliza a edição do campo (salva e sai), sem navegar para outro campo, sem abrir o próximo e sem entrar em outra meta — mesmo comportamento já garantido para a tecla Tab física desde a vBeta1.6.
- Nenhuma outra tela, cálculo ou fluxo foi alterado.

Status atual:
- Pronto para uso. Recomenda-se testar no celular: tocar em Descrição, digitar, tocar em "Concluído"; tocar em Valor, digitar, tocar em "Concluído". Em nenhum dos dois casos o foco deve mudar para outro campo.

MetaDriver vBeta1.7

Solicitado:
- Corrigir vazamento de dados entre usuários ao trocar o login no mesmo celular.

Feito:
- Identificada a causa: os dados eram carregados de uma chave local global (não isolada por usuário) antes do escopo do usuário ser ativado, porque essa ativação só acontecia depois, de forma assíncrona, no fluxo de login com a nuvem.
- O escopo do usuário agora é ativado de forma síncrona, antes de qualquer leitura de dados, sempre que já existe um usuário em cache no aparelho.
- A chave global é apagada depois de migrada para a chave isolada do usuário, para não ser herdada por outra conta no futuro.
- Chaves de dados antigas (de antes do sistema multiusuário) deixaram de ser usadas como fallback quando já há um usuário autenticado no aparelho.
- Nenhuma outra tela, cálculo ou fluxo foi alterado.

Status atual:
- Pronto para uso. Recomenda-se testar login com duas contas diferentes no mesmo aparelho para validar.
